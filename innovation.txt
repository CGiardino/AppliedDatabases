Docker Compose and Seeding:
The `resources/docker-compose.yml` file starts the MySQL and Neo4j services used by the application.
MySQL loads the schema and sample data from `appdbproj.sql` when the container starts.
Neo4j uses `NEO4J_AUTH` for authentication and a healthcheck to wait until the database is ready.
The `neo4j-seed` service runs `resources/neo4j_seed.sh` after Neo4j becomes healthy.
The seeding script imports data from `appdbprojNeo4j.json` into Neo4j.
If the Neo4j data volume is removed, the database is reinitialized and reseeded on the next startup.
